# Mobs Authority v1

## Goal

v1 replaces the old "every peer owns real mobs and we repair drift" model with a single-authority mob world.

The target invariant is:

- Host owns every real `dc.en.Mob`.
- Clients do not simulate authoritative mob AI, damage, death, drops, elite state, or spawn decisions.
- Clients render remote mobs as non-authoritative proxies.
- Client attacks become requests to the host.
- The host performs hit validation and broadcasts results.

This avoids the v0.3.1 class of bugs where both sides have different local mob state, elite variants, spawn order, or death timing.

## Non-Goals

- Do not try to synchronize every mutable field of local mobs across peers.
- Do not use `level.entities` index as a long-lived network identity.
- Do not make visual proxies attackable through local collision.
- Do not let clients decide mob death, loot, or elite modifiers.

## Network Identity

Every host mob gets a stable `netMobId`.

`netMobId` must be generated by the host and remain stable until despawn. It must not depend on `level.entities` order.

Recommended identity mapping:

```text
Host:
  Mob object reference -> netMobId
  netMobId -> Mob object reference

Client:
  netMobId -> visual proxy
```

## Wire Model

The v1 protocol should be event plus snapshot based:

```text
MOBV1_SPAWN
  hostUserId
  levelId
  netMobId
  type
  elite/variant metadata
  seed if available
  initial x/y
  maxHp
  initial anim

MOBV1_STATE
  hostUserId
  levelId
  netMobId
  x/y
  velocity if available
  dir
  hp/maxHp
  anim/group
  flags

MOBV1_DESPAWN
  hostUserId
  levelId
  netMobId
  reason

MOBV1_HIT_REQUEST
  attackerUserId
  levelId
  netMobId
  weapon/action info
  attack timestamp
  attack id
  hero x/y/dir
  sampled target x/y
  hit radius

MOBV1_HIT_RESULT
  hostUserId
  attackerUserId
  levelId
  netMobId
  accepted/rejected
  damage
  hp after hit
  death flag
  reason
  fx info if available
```

## Stages

### Stage 1: Host-authority visuals

- Keep current projection layer as a temporary visual proxy implementation. Done in `MobProjectionLayer`.
- Change identity from local entity index to stable `netMobId`. Done in `MobAuthorityV1Ids`.
- Host broadcasts all authoritative mobs through `MOBV1STATE`. Done.
- Clients only display host mobs as proxies. Done for `MOBV1STATE`.
- Host emits explicit `MOBV1SPAWN` and `MOBV1DESPAWN`. Done.
- Host may optionally display client-only projections while client mob spawning is not yet suppressed.

### Stage 2: Client mob suppression

- Prevent or neutralize client-created real mobs for the shared run. The first neutralization pass is implemented in `MobAuthorityV1ClientSuppression`.
- Client mobs are neutralized after `Level.entitiesPostCreate` and again each hero update, so local mob AI/damage paths are suppressed as early as the current hook surface allows.
- Client levels should still be playable, but local mobs must not be damage authorities.
- Client local mob `contactAttack`, `onTouch`, and `queueAttack` are suppressed in v1 mode.
- Client local mob update/damage/death paths are suppressed in v1 authority mode; velocity is zeroed and the mob is hidden/untargetable.
- Visual proxies must stay outside `level.entities`.

### Stage 3: Host hit validation

- Client attack sends `MOBV1HITREQ`. Nearest-proxy request dispatch is implemented through `NetNode.SendAttack`.
- The request now carries `attackId`, send timestamp, hero x/y/dir, sampled target x/y, hit radius, action token, and damage hint.
- Host validates against real mobs and host-side positions. `netMobId`, level/life, hit radius, direction, vertical range, and replay-window validation are implemented.
- Host applies a minimal authoritative damage path after validation. Damage is clamped on the host and returned structurally in `MOBV1HITRES`; full vanilla `AttackData` replay is still pending.
- Host broadcasts `MOBV1HITRES` with accepted/rejected, damage, hp after hit, death flag, and reason.
- Client plays visual feedback only after host result; accepted non-lethal hits flash the projection and lethal hits remove it immediately.

### Stage 4: Spawn and elite consistency

- Host broadcasts spawn and elite metadata. Basic spawn/despawn events are implemented; elite metadata is still pending.
- Clients create proxies from host metadata only.
- Despawn is explicit; stale timeout remains a safety net, not the normal path. Basic despawn is implemented.

## Compatibility Rule

`EnableMobsSync` and `DebugModuleId.MobsSynchronization` select the mob sync mode:

- Both enabled: v1 host-authority mob sync. Host owns real mobs, clients render host proxies, client attacks become host-validated hit requests.
- Both disabled: projection-only mode. Peers exchange mob projections so each side can see the other's mobs, but projections are non-interactive and cannot be attacked.
- Mixed states: mob v1/projection transport stays disabled.

These switches must not re-enable the old v0.3.1 full mob sync path; during migration, v1 code remains isolated so the old hook-heavy module can be removed or left disabled.
